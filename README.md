# کیف پول سازمانی (نسخه اولیه)

این مخزن شامل قدم نخست برای پیاده‌سازی سامانهٔ میکروسرویسی «کیف‌پول سازمانی» است. در این مرحله ساختار پایه و سرویس هویت با لایه‌های تفکیک‌شده ایجاد شده است.

## آنچه در این گام انجام شد

- ایجاد ساختار Solution `EnterpriseWallet` همراه با پوشه‌بندی سرویس‌ها، بلوک‌های سازنده و پوشهٔ مستندات.
- پیاده‌سازی اولیهٔ سرویس هویت با لایه‌های `Api`، `Application`، `Domain` و `Infrastructure`.
- تنظیم Swagger فارسی، محلی‌سازی و یک نمونه نقطهٔ پایانی برای ثبت کاربر.
- تعریف موجودیت دامنهٔ «کاربر» و الگوی اولیهٔ کار با پایگاه‌داده و واحدکار.

## گام‌های بعدی پیشنهادی

1. تکمیل عملیات احراز هویت و مدیریت نقش‌ها با JWT و ذخیره‌سازی ایمن رازها.
2. اضافه کردن تست‌های واحد و یکپارچه برای سناریوهای هویت.
3. افزودن سایر سرویس‌ها (Wallet، ضدتقلب، تسویه و ...).
4. آماده‌سازی Docker Compose و مانیفست‌های Kubernetes.

> تمامی نام‌گذاری‌ها و پیام‌ها در این مرحله به‌صورت فارسی نوشته شده‌اند تا با راهبرد بومی‌سازی سامانه همسو باشند.

## راهنمای تنظیم ریموت GitHub

فرمان‌های Git باید در **ترمینال (Command Prompt / PowerShell / Git Bash / Terminal VS Code)** اجرا شوند؛ GitHub خود وب‌سایتی برای اجرای این دستورها ندارد. بنابراین ابتدا روی رایانهٔ خود یا محیط توسعه‌ای که مخزن را در آن کلون کرده‌اید (مانند GitHub Codespaces یا Dev Container)، ترمینال را باز کنید و وارد شاخهٔ ریشهٔ پروژه شوید.

مثلاً اگر مخزن را در مسیر `~/Projects/wallet` کلون کرده‌اید، ابتدا وارد آن شوید:

```bash
cd ~/Projects/wallet
```

در این مرحله می‌توانید ریموت را اضافه کنید:

1. در GitHub یک **Personal Access Token** با سطح دسترسی `repo` بسازید و مقدار آن را یادداشت کنید.
2. از همین مسیر ریشهٔ مخزن، فرمان زیر را اجرا کنید و مقادیر داخل کروشه را با اطلاعات خودتان جایگزین کنید:

   ```bash
   git remote add origin https://[USERNAME]:[TOKEN]@github.com/[USERNAME]/[REPOSITORY].git
   ```

   اگر قبلاً ریموتی تعریف شده، از `git remote set-url origin ...` استفاده کنید.
3. با دستور `git remote -v` از ثبت شدن ریموت مطمئن شوید و در صورت نیاز با `git push --dry-run origin work` ارتباط را تست کنید.
4. پس از اطمینان، با `git push origin work` (یا نام شاخهٔ مدنظر) تغییرها را به GitHub منتقل کنید.

> توصیه می‌شود قبل از push، با `git status` و `git log --oneline` وضعیت مخزن را بررسی کنید.

## وضعیت کنونی مخزن

- شاخهٔ فعال محلی: `work`
- ریموت `origin` به مخزن `shahinmirzaei/wallet` متصل شده است.
- آخرین تغییرات موجود (ساختار سرویس هویت و فایل‌های پایه) با دستور `git push -u origin work` به ریموت ارسال شدند؛ بنابراین مخزن GitHub دیگر خالی نیست و تاریخچهٔ مربوطه در دسترس است.
